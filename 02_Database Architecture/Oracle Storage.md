## Control Files

Save the databased physical structure:
- Database file names and their locations
- Backup files
- Database basic information like creation time and last checkpoint

Oracle database instance needs only a single control file to operate, most production databases, more than one control files are configured.

## Parameter File

A small file that saves key-value pairs, which are called **Initialization Parameters**, it is a configuration parameters that control the instance behavior.

## Online Redo Log Files

Stores the database changes, the redo entries.

## System Data Files

Data files which contain data used by Oracle internal operations.

## Automatic Diagnostic Repository (ADR)

A directory structure where the database saves data such as trace files, alert log files, DDL log files, and the health monitor reports.

## User Tablespace Data Files

Files that contain user data, those files contain the data that the RDBMS is built for.

## Backup Files

Used to restore the database in case of a data losses.

## Archive Redo Log Files

Offline copies of the online redo log files when they are filled by the redo entries.

## Password File

Store the passwords of the database users with the administrative privileges.

## Keystore (Wallet)

A file that stores encryption keys used by the database.

## Block Change Tracking (CBT) Files

Used when BCT is enabled, a feature that speeds incremental backups.

## Flashback Logs

Files that get generated by the database when the flashback option is enabled.
